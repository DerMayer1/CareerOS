# Workflow: Report

```bash
career-os report
career-os show top
career-os show gaps
career-os show red-flags
career-os explain <job_id>
```

Reports should not reprocess descriptions or spend tokens. They should render from the current normalized/scored data.

The report should answer:

- which jobs are strongest
- why they are strongest
- what to review manually
- what to skip and why
- which skill gaps repeat
- which red flags are blocking
- what the next actions are
